# Trust402 Brand Direction: x402 Radar

This folder captures the selected brand direction for Trust402: a live radar for x402 resources.

The goal is to make Trust402 feel like an operational trust layer for the agent economy: it discovers resources, compares price and risk, watches availability, and routes payments only through policy-controlled paths.

## Positioning

Trust402 is not a generic assistant. It is a buyer-side procurement and trust agent for x402 resources.

Core promise:

> Track endpoints, compare prices, detect risk, and route safe payments through policy.

## Visual Idea

The selected direction is `x402 Radar`.

It should feel:

- active, but controlled
- technical, but understandable
- marketplace-aware, but not speculative
- autonomous, but visibly bounded by policy
- premium SaaS, not crypto-hype

## Primary Screens

- Landing hero with radar/resource grid
- Resource comparison cards
- Policy gate / spend limit panel
- Receipt and proof trail
- Live resource status table

## Files

- `design-system.md` - visual rules, layout, UI motifs, and usage guidance
- `landing-copy.md` - page copy and section structure for the x402 Radar direction
- `landing-preview.html` - clean production landing source for the selected direction
- `tokens.json` - machine-readable color, typography, spacing, and component tokens
- `tokens.css` - CSS custom properties for quick implementation
- `assets/` - visual references and future generated assets
